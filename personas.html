Payment Company · ผู้ให้บริการชำระเงิน

😣 Pain Points

  • Transaction ค้างเมื่อ network หลุด
  • ลูกค้า dispute ยอดเงินผิด
  • ต้อง reconcile ทุกวัน

🎯 Goals

  • อัปเดต payment status ได้ real-time
  • ลดขั้นตอน manual intervention
  • audit trail ครบถ้วน

Roleplay Scenario — ทุกครั้งที่มี Order ใหม่
01
ประมวลผลการชำระเงิน QR PromptPay
Automated Flow · เกิดขึ้น <3 วินาที
1
รับ Payment Request จาก Bookshop API
order_id, amount=379, method=promptpay, user_id
2
Read Payment Order → ตรวจสอบข้อมูล
validate ยอดเงิน, เช็ค duplicate transaction, verify user identity
3
Generate QR Code → ส่งกลับให้ลูกค้า
QR หมดอายุใน 10 นาที ลูกค้าสแกนจ่าย
4
ยืนยันรับเงิน → Update Payment Status = "SUCCESS"
ส่ง webhook กลับ Bookshop ระบบ Bookshop update order status เป็น "paid" ทันที
⚠️ กรณีพิเศษ: ถ้าลูกค้าสแกนแล้วเงินหักแต่ webhook ส่งไม่ถึง — FinPay จะ retry 3 ครั้ง ถ้ายังไม่ได้จะ flag เป็น "PENDING_REVIEW" ให้ Ops team เช็คและ Update Payment Status ด้วยตัวเอง
